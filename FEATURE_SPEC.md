# Ide/AI — Feature Specification

## Feature 1: Idea Input & Onboarding
**Component**: `pages/Home.tsx` + `components/nebula/IdeaNebulaCanvas.tsx`
- Single centered text input with placeholder "Describe your product in one sentence…"
- Below input: 4 selector chips (Platform, Audience, Complexity, Tone)
  - Platform options: Bubble, Webflow, FlutterFlow, Bolt, Lovable, Claude Code, Cursor, Replit, n8n, Custom
  - Audience: Consumers, Businesses, Internal Team, Developers
  - Complexity: Simple (1–5 screens), Medium (5–15), Complex (15+)
  - Tone: Formal, Casual, Technical, Startup-style
- Background canvas: 8 faint nodes connected by thin lines, subtle pulse animation via Framer Motion
- On submit → create project in DB → navigate to /discovery/:projectId

## Feature 2: Discovery Phase (SSE Chat)
**Component**: `pages/Discovery.tsx` + `components/discovery/`
- Left: Stage stepper (5 stages), vertical
- Center: Chat thread with streamed AI responses
- Right: Live Design Sheet panel (updates via Zustand as AI extracts info)
- AI follows a Socratic state machine (see Architecture Doc)
- After each AI message, backend updates design_sheet record and emits a `sheet_update` SSE event
- Confidence score recalculates on each sheet update
- Quick reply chips generated by AI per turn

## Feature 3: Design Blocks Board
**Component**: `pages/Blocks.tsx` + `components/blocks/`
- Drag-and-drop grid using @dnd-kit/core
- Each block card: title, description, MVP/V2 toggle, effort badge (S/M/L), delete button
- AI auto-generates initial blocks from completed design sheet
- Scope slider at top: Lean / Balanced / Full (filters visible blocks)
- Right panel: Prompt Kit Preview — live preview of how these blocks translate to prompts for chosen platform
- "Regenerate blocks" and "Add custom block" actions

## Feature 4: Pipeline Builder
**Component**: `pages/Pipeline.tsx` + `components/pipeline/`
- Horizontal canvas with 7 layer cards: Frontend, Backend, Database, Automations, AI/Agents, Analytics, Deployment
- Each card has a tool dropdown with curated options per layer
- AI recommends initial pipeline based on project type from design sheet
- Right panel: estimated monthly cost range, complexity score, "Why this stack" bullets
- "Generate UI Skeleton" button → triggers export_service to produce screen list JSON
- "Swap" action on each card triggers AI to re-evaluate compatibility

## Feature 5: Prompt Kit Generator
**Component**: `components/promptkit/PromptKitPanel.tsx`
- Displays structured, copyable prompt blocks for selected platform
- Sections: System Context, App Description, Feature List, Data Model, Constraints, First Task
- Toggle: Technical / Zero Dev Language Mode
- "Copy all", "Copy section", "Regenerate" actions
- Saved automatically to project's prompt_kits table

## Feature 6: Project Folder System
**Component**: `components/projects/FolderTree.tsx`
- Left sidebar persistent folder tree
- Sections per project: Discovery Notes, Design Sheet, Prompt Kit, Pipeline Map, Exports, Versions
- Versions shown as timeline dots at folder bottom; click to restore snapshot
- "New Project", "Duplicate", "Archive" actions

## Feature 7: Pitch Mode
**Component**: `pages/PitchMode.tsx`
- Clean single-column document: Title, Value Prop, Audience, Features (3–5), MVP Scope, Simple Flow Diagram
- Flow diagram: React Flow (read-only) with 4–6 nodes
- Share panel: toggle public link, copy URL, set expiry
- Print/PDF export via browser print API

## Feature 8: Export System
**Router**: `backend/app/routers/exports.py`
**Service**: `backend/app/services/export_service.py`
- Formats: .md, .txt, .pdf (fpdf2), .docx (python-docx)
- Platform-specific Jinja2 templates per format
- ZIP download option (all formats at once)
- Frontend: Export page with format selector, preview pane, download button
