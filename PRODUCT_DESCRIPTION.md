# Ide/AI — Product Description

## What it is

Ide/AI is a web application that takes a rough product idea and turns it into a complete, structured design kit — ready to hand off to any no-code or low-code builder platform. It replaces the expensive, time-consuming planning phase that typically happens *inside* builder tools with a dedicated, guided process that happens *before* you ever open one.

## The problem it solves

Most no-code and low-code platforms charge by usage — credits, AI interactions, compute time. The planning and iteration that inevitably happens when building something new burns through those credits fast. Users spend significant money just figuring out *what* to build: going back and forth with AI assistants, restructuring features, changing scope, rethinking data models. By the time the actual building starts, a large portion of the budget has already been spent on decisions that could have been made beforehand.

Ide/AI moves that entire planning phase out of the builder and into a purpose-built environment where users can take as long as they need to get their thinking right — without watching a credit meter tick down.

## How it works

The process takes roughly 15 to 30 minutes from start to finish:

### 1. Describe the idea

The user enters a one-sentence product description and selects their target builder platform (Bubble, Webflow, FlutterFlow, Bolt, Cursor, Replit, Claude Code, and others), intended audience, expected complexity, and preferred tone.

### 2. AI-guided discovery

A structured conversation walks the user through five stages — clarifying the core problem, defining the target audience, identifying features, establishing technical constraints, and confirming the complete picture. The AI asks focused questions rather than open-ended ones, and suggests quick-reply options at each step so the user is never stuck staring at a blank input field.

As the conversation progresses, a live design sheet fills in automatically on the side panel. A confidence score tracks how complete the design specification is, recalculating after every exchange based on how thoroughly each area has been covered.

### 3. Feature decomposition

Once discovery is complete, the system generates a board of 8 to 12 feature blocks — individual, scoped pieces of functionality with effort estimates, priority levels (MVP or post-launch), and categories. Users can adjust these freely: toggle priorities, change effort ratings, remove blocks they don't need, or add custom ones. A scope slider filters the view between lean (minimal viable), balanced (all MVP features), and full (everything including post-launch items).

### 4. Tech stack recommendation

A pipeline builder lays out seven infrastructure layers — frontend, backend, database, automations, AI/agents, analytics, and deployment — and recommends specific tools for each based on the project's requirements. Each recommendation includes reasoning and a monthly cost estimate. Users can swap individual tools and the system flags compatibility issues between choices.

### 5. Platform-specific prompt kits

This is where the output becomes directly actionable. The system generates detailed, structured prompts tailored to the user's chosen builder platform. A Bubble prompt kit includes data type definitions and workflow instructions. A Claude Code prompt kit includes database schemas, API endpoint specifications, and a phased implementation plan. A Bolt prompt kit is formatted as a single paste-ready prompt with follow-up prompts for iterative refinement. Seven platform templates are available, each written in the format and terminology that platform expects.

### 6. Export and share

The complete design kit — design sheet, feature blocks, tech stack, and everything else — can be downloaded as Markdown, plain text, PDF, Word document, or a ZIP bundle containing all formats. A pitch mode generates a clean, one-page project brief suitable for sharing with stakeholders or collaborators, with print-optimized formatting.

### 7. Version history

The system auto-saves snapshots at key milestones. Users can revisit and restore any previous version of their project, making it safe to experiment with different directions without losing prior work.

## What the user walks away with

A complete design kit containing:

- A clearly articulated problem statement, audience definition, and MVP scope
- A prioritized, effort-estimated feature breakdown
- A recommended tech stack with cost projections
- Platform-specific prompts ready to paste directly into their chosen builder tool
- A shareable one-page brief for anyone else involved in the project
- Downloadable documentation in multiple formats

## Why this matters

The conventional approach to building with no-code and low-code tools is to open the platform, start talking to its AI assistant, and figure things out as you go. This works for simple projects, but anything with real scope means dozens or hundreds of AI interactions just to settle on a direction — each one consuming credits on platforms where those interactions are limited and expensive.

Ide/AI gives users a space to do all of that thinking first, in a tool designed specifically for the planning stage. The AI interactions are focused and structured rather than open-ended, which means users reach clarity faster. And because the output is formatted for the specific platform they plan to use, there's no translation step — they paste the prompt kit into their builder and start executing immediately against a well-defined plan.

The result is that when users do open their builder tool, they already know exactly what they're building, how the features break down, what their data model looks like, and what order to build things in. They spend their builder credits on *building*, not on planning.
